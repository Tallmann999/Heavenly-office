# Проект (рабочее название)

Рабочее название: Божественная канцелярия (Divine Office)

**Жанр и формат сессий:** 2D/2.5D кабинетной симуляции с короткими сессиями (5–15 минут).

**Целевая аудитория и ключевое обещание:**
- ЦА: казуально‑инди игроки, любители коллекционирования и сюжетных микро‑решений (18–45 лет).
- Обещание: принимать моральные решения, наблюдать за их последствиями и собирать уникальные карты душ.

**Core loop (кратко):**
- Очередь душ → Открыть досье → Выбрать печать → Активировать рычаг → Показ результата → Получить награду/карту → Сохранить прогресс → Следующая душа.

**Прогрессия (кратко):**
- Короткий горизонт: обработка смены (5 душ), получение карт и очков.
- Средний: сбор коллекции карт и накопление ресурсов офиса.
- Длинный: улучшения отдела, разблокировка новых типов дел и миров.

**Модель дистрибуции:** premium / локальный прототип; в MVP нет IAP (позже P1/P2 — возможен hybrid).

**Границы MVP:**
- Включено: основной flow (5 душ подряд), PF_DivineOfficeMainScreen, PF_ReincarnationResultScreen, Card Collection экран (прототип), 5 Soul SO, 8 Reincarnation SO, сохранение (module_json), RU/EN локализация, placeholder UI.
- Не включено: финальная нарезка арта, озвучка, полноценная экономика, Steam/онлайн‑сервисы.

**Технологические решения (подтверждённые):**
- Unity LTS: 2022.3.
- Сохранение: `DivineOfficeSaveService` — JSON в `Application.persistentDataPath` с fallback на `PlayerPrefs` (`module_json`).
- Локализация: SO‑таблицы RU/EN + `LocalizedTextBinder`, переключение языка в рантайме.
- UI: расширять существующий `HeavenOffice` через адаптер; использовать ранее сгенерированные prefabs (`use_prefabs`).

**Ключевые риски:**
1. Интеграция с `HeavenOfficeView` — возможные регрессии; стратегия: адаптер и минимальные изменения, покрыть smoke‑тестами.
2. Сохранение — конфликт с потенциальной существующей save‑системой; стратегия: предусмотреть адаптер/интеграцию при обнаружении проектного SaveService.
3. Локализация — отсутствие корректных TMP font assets для кириллицы; стратегия: проверить TMP font assets и добавить fallback.

**Acceptance criteria (MVP):**
- Запускается сцена прототипа, показывается первая душа из SO.
- Можно пройти 5 душ подряд: выбрать печать, активировать рычаг, увидеть результат.
- Выдается и сохраняется хотя бы одна карта при правильном решении.
- RU/EN переключение обновляет UI без перезапуска сцены.
- Сохранение сохраняет день, открытые карты, язык и обработанные души.
