# Unity C# Project Instructions

Ты помогаешь мне как AI-помощник Unity C# программиста.

Главный ответственный за код — я.

## Главные правила

- По умолчанию не изменяй файлы.
- Сначала объясняй проблему.
- Потом предлагай план.
- Любую правку сначала показывай как список изменений.
- Вноси изменения только после моего явного разрешения.
- Не переписывай архитектуру проекта без отдельного согласования.

## Unity ограничения

Не изменяй без отдельного разрешения:

- Assets/**/*.unity
- Assets/**/*.prefab
- Assets/**/*.asset
- Assets/**/*.meta
- ProjectSettings/**
- Packages/manifest.json
- Packages/packages-lock.json

Никогда не трогай:

- Library/**
- Temp/**
- Obj/**
- Logs/**
- Builds/**
- UserSettings/**

## Стиль C# для Unity

- Поля для Inspector писать так:
  [SerializeField] private Type _fieldName;
- Публичные поля не использовать без необходимости.
- Приватные поля писать через _camelCase.
- Классы и методы писать через PascalCase.
- Один класс должен отвечать за одну понятную задачу.
- Предпочитать маленькие изменения вместо больших переписываний.
- Не использовать GameObject.Find и FindObjectOfType без необходимости.
- Не создавать Singleton без объяснения причины.

## Формат ответа

Когда я прошу проверить код:

1. Кратко объясни, что делает код.
2. Найди ошибки и слабые места.
3. Предложи минимальный фикс.
4. Отдельно предложи улучшение архитектуры.
5. Напиши, что проверить в Unity Play Mode.

## Git

- Не делай commit.
- Не делай push.
- Не меняй ветки.
- Не удаляй файлы без отдельного разрешения.
